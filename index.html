AMIE : Application Mobile pour les Intervenants de l'Environnement

Ce projet a été réalisé par des étudiants de l'ENSG (École Nationale des Sciences Géographiques) dans le cadre du PDI (Projet de Développement Informatique) des élèves ingénieurs de 2ème année et Master première année. La structure commanditaire est la Direction régionale Centre-Val de Loire de l’OFB (Office Français de la Biodiversité) Il a pour objet la création d’une application mobile hors connexion pour une visualisation de données cartographiques nécessaires aux actions de terrain des inspecteurs de l’environnement. Deux éléments logiciels composent la solution retenue : - un outil (Tuilator) permettant de générer des jeux de données sous forme de tuiles vectorielles et raster. - une application mobile (BioViz) pour la consultation des données sur le terrain

BioViz : l’application mobile, web progressive

Spécifiquement paramétrée pour une consultation rapide de données en étant géolocalisé, cette application offre une navigation fluide sur tout jeu de donnée préalablement chargé.

Visualiser

Tout jeu de données vecteur ou raster peut-être visualisé avec une sémiologie définie dans un projet central sous Qgis, le logiciel SIG qui équipe les agents de l’OFB.

Hors ligne

Fréquemment hors couverture des réseaux de téléphonie mobile, la consultation hors ligne est indispensable aux agents. L’élection d’emprises adaptées à leur périmètres d’intervention permet d’optimiser le stockage de données haute résolution (raster) ou volumineuses (données d’observations naturalistes ou sur la continuité écologique)

Localiser

L’utilisateur accède à sa position GPS pour localiser tous les zonages et point d’intérêt à proximité immédiate.

Mesurer

Plusieurs dispositions réglementaires sont dépendantes du dimensionnements d’éléments naturels ou artificiels (plans d’eau, remblais…). La mesure de terrain rapide peut constituer un outil d’aide à la décision.

SmartTuileur : Générateur de jeux de données tuilés

Mettre en forme

L’intégration du composant logiciel à QGIS permet de préparer les données avec toutes les modalités offertes par ce logiciel.

Dimensionner

La volumétrie des fichiers raster aux niveaux de zoom élevée implique des contraintes de stockage et de vitesse d’affichage qui sont traitées par un dimensionnement d’emprise spatiale adapté aux périmètres d’intervention des agents (calé sur les grilles de l’INPN (Inventaire national du Patrimoine Naturel))

Multiplier

Les modalités de sélection multiple des couches de données permettent la production en une seule opération d’un contexte régional multi-thématique.

Partager

La fabrication des tuiles peut bien sûr faire l’objet d’une mutualisation sur un dépôt partagé entre agents de même secteur, économisant ainsi ressource logicielle, temps de fabrication et échange de bonnes pratiques pour la sémiologie.
